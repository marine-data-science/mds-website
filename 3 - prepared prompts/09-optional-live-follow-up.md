# 09 - Optional Live Follow-Up

## Purpose

Use this as a small live moment after showing the prepared recording. It should be low-risk and reviewable within a few minutes.

Pick one variant, not all of them.

## Option A - Improve Thesis Call-To-Action

```text
Please improve the thesis-related call-to-action on the homepage and theses page.

Goal:
- Make it clearer for students what they can do next.
- Keep the tone academic and direct.
- Do not invent thesis topics or requirements.
- Use existing content from `1 - content/theses/`.

Make the smallest useful change, run the build, and summarize the diff.
```

## Option B - Accessibility Review

```text
Please do a focused accessibility review of the homepage only.

Check:
- heading order
- link text
- image alt text
- color contrast
- keyboard-visible focus states
- mobile readability

Make small fixes only, run the build, and summarize what you changed.
```

## Option C - Content Clarity

```text
Please improve the clarity of the Research overview page without changing the scientific meaning.

Goal:
- Make the page easier for students and external visitors to scan.
- Keep original source content and source URLs intact.
- Do not invent claims.
- Prefer headings, summaries, tags, or layout improvements over rewriting everything.

Run the build and summarize the changes.
```

## Review checkpoint

This optional step should show that live interaction does not need to be huge. A small, well-reviewed redirect is often more educational than a risky end-to-end build.
