# 04 - Homepage First Slice

## Purpose

Use this to produce the first visually meaningful page. This should be the "now we can see something" moment in the demo.

The scope is intentionally one page: homepage, navigation, and a few real content previews.

## Copy-paste prompt

```text
Please build the first visible website slice: the homepage and global navigation.

Use:
- `2- styling/design-direction.md` as the design brief.
- `2- styling/Logo_MDS_1-3-Final_color.png` or the SVG logo if technically preferable.
- Real content from `1 - content/`, especially Research, Projects, People, and Theses.

Homepage goals:
- Immediately signal "Marine Data Science".
- Feel modern, academic, maritime/marine, minimal, and student-friendly.
- Show real content above or near the first fold, not just a generic hero.
- Include clear navigation to Research, Projects, People, Theses, Teaching, and Publications.
- Preview at least:
  - selected research topics
  - selected projects
  - thesis opportunities
  - people or contact information

Design constraints:
- Do not create a generic SaaS landing page.
- Avoid playful decoration and stock-photo atmosphere.
- Use logo-derived blues and strong typography.
- Keep text readable and layouts responsive.
- Use local assets only.

Verification:
- Run the build.
- If a dev server or preview is available, use it to inspect the homepage visually.
- Report what you would improve in the next iteration.
```

## Review checkpoint

Pause the video and ask:

- Does this look like a research group website?
- Is the marine/data-science identity visible?
- Is the content real?
- What would a human designer or PI reject?
- What should we ask the agent to improve next?

## Good outcome

A homepage exists with credible identity, real content, and a clear direction. It may still need polish.
